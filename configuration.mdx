## Brand Color and Theme Adjustments

Set your brand color to `#3B82F6` for a professional blue theme. Use the UI or API to apply changes.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to `https://dashboard.example.com/settings/theme`.
  </Step>
  <Step title="Update Color" icon="palette">
    Enter `#3B82F6` in the brand color field and save.
  </Step>
  <Step title="Preview Changes" icon="eye">
    View live preview and publish.
  </Step>
</Steps>

<CodeGroup tabs="JSON,YAML">
  ```json
  {
    "theme": {
      "brandColor": "#3B82F6",
      "mode": "light"
    }
  }
  ```
  ```yaml
  theme:
    brandColor: "#3B82F6"
    mode: light
  ```
</CodeGroup>

<Callout kind="tip">
  Changes apply site-wide within `<1min`.
</Callout>

## User Access and Permissions Setup

Define roles like Admin, Editor, and Viewer. Assign via dashboard or API.

| Role     | Read | Write | Admin |
|----------|------|-------|-------|
| Admin    | ✅   | ✅    | ✅    |
| Editor   | ✅   | ✅    | ❌    |
| Viewer   | ✅   | ❌    | ❌    |

<ParamField path="user.role" param-type="string" required="true">
  Role assignment: `admin`, `editor`, or `viewer`.
</ParamField>

<ParamField header="X-API-Key" param-type="string" required="true">
  Use `YOUR_API_KEY` for permission updates.
</ParamField>

## Backup and Export Configurations

Schedule automated backups to prevent data loss.

<Tabs>
  <Tab title="Daily Backup" icon="calendar">
    ```json
    {
      "backup": {
        "schedule": "0 2 * * *",
        "format": "zip",
        "destination": "s3://your-bucket/backups"
      }
    }
    ```
  </Tab>
  <Tab title="Manual Export" icon="download">
    <Steps>
      <Step title="Select Pages">
        Choose docs to export.
      </Step>
      <Step title="Generate">
        Click Export MDX bundle.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Integration with External Tools

Connect GitHub, Slack, or webhooks.

<Expandable title="Advanced GitHub Sync" default-open="false">

Configure webhook at `https://your-webhook-url.com/webhook`.

```javascript
fetch('https://api.example.com/integrations/github', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' },
  body: JSON.stringify({
    repo: 'your-org/docs',
    branch: 'main'
  })
});
```

</Expandable>

<Callout kind="alert">
  Test integrations in staging first.
</Callout>
