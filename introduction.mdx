## Overview

Welcome to Aleksandr Kislitsyn's documentation platform. You get a centralized space to organize, manage, and share all your project documentation. Build structured guides, embed code examples, and customize branding with ease using intuitive MDX components.

This default space helps you create professional docs without leaving your workflow. Start with pre-built components like Cards, Steps, and Tabs to showcase features and guides quickly.

## Quick Start

Get up and running in minutes:

<Steps>
  <Step title="Create Your First Page" icon="file-plus">
    Add a new `.mdx` file to your docs directory.

    ```
    touch docs/quickstart.mdx
    ```
  </Step>
  <Step title="Add Frontmatter" icon="edit-3">
    Start every page with YAML frontmatter for title and description.
  </Step>
  <Step title="Use Components" icon="code">
    Insert JSX components like `<Card>` for rich layouts.
  </Step>
  <Step title="Preview Changes" icon="eye">
    Run your dev server to see live updates.
  </Step>
</Steps>

<Callout kind="tip">
  Begin with the `/quickstart` page for detailed setup instructions.
</Callout>

## Customize Your Brand

Tailor the documentation to match your identity, starting with the brand color `#3B82F6`.

<Tabs>
  <Tab title="Theme Config" icon="settings">
    Update your theme file:

    <CodeGroup tabs="JavaScript,TypeScript">
    ```javascript
    // theme.config.js
    export default {
      colors: {
        primary: '#3B82F6'
      }
    };
    ```
    ```typescript
    // theme.config.ts
    const config = {
      colors: {
        primary: '#3B82F6'
      }
    } as const;
    export default config;
    ```
    </CodeGroup>
  </Tab>
  <Tab title="CSS Variables" icon="code">
    Override styles globally:

    ```css
    :root {
      --brand-color: #3B82F6;
      --brand-hover: #2563EB;
    }
    ```
  </Tab>
</Tabs>

<Callout kind="info">
  Preview changes in your dev environment. Test across light and dark modes for consistency.
</Callout>
